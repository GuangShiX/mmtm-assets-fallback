# MementoMori Critical Icon Fallback

Small, versioned fallback set of MementoMori icons used by
[`mementomori-helper`](https://github.com/GuangShiX/mementomori-helper-GXmaster) and generated by
[`mmtm-apk-tracker`](https://github.com/GuangShiX/mmtm-apk-tracker).

This repository is a disaster-recovery source for low-traffic use when the
primary OSS/CDN is unavailable. It is not the primary CDN and does not contain
APKs, complete character artwork, raw Unity objects, or databases.

## Contents

```text
assets/
  characters/   # Small character portraits
  enemies/      # Enemy portraits
  equipment/    # Equipment icons
  spheres/      # Rune/sphere icons
  items/        # Common item icons
  ui/           # Nine-slice frames, decorations, and element icons
manifest.json   # Paths, sizes, and SHA-256 hashes for the tagged version
latest.json     # Current immutable tag and manifest URL
```

Consumers should request
[`latest.json`](https://raw.githubusercontent.com/GuangShiX/mmtm-assets-fallback/main/latest.json),
download the tagged `manifest_url`, and cache each selected file locally after
verifying its SHA-256. Asset URLs are built as `{base_url}/{path}`.

Do not construct URLs against the moving `main` branch. The manifest points to
an immutable game-version tag such as `v4.18.0`.

## Updating

The scheduled workflow checks APKPure every six hours. It performs a complete
tracker extraction only when the game version changes, rebuilds the critical
set, commits it, and creates an immutable `v<game-version>` tag.

# MementoMori Critical Icon Fallback

Small fallback set of MementoMori icons used by
[`mementomori-helper`](https://github.com/GuangShiX/mementomori-helper-GXmaster) and generated by
[`mmtm-apk-tracker`](https://github.com/GuangShiX/mmtm-apk-tracker).

This repository is a disaster-recovery source for low-traffic use when the
primary Cloudflare R2 source is unavailable. It is not the primary CDN and does not contain
APKs, complete character artwork, raw Unity objects, or databases.

## Contents

```text
assets/
  characters/   # Small character portraits
  enemies/      # Enemy portraits
  equipment/    # Equipment icons
  spheres/      # Rune/sphere icons
  items/        # Common item icons
  ui/           # Nine-slice frames, decorations, and element icons
manifest.json   # Stable paths, sizes, and SHA-256 hashes
latest.json     # Current stable manifest URL
```

Consumers should request
[`manifest.json`](https://raw.githubusercontent.com/GuangShiX/mmtm-assets-fallback/main/manifest.json)
and cache each selected file locally after verifying its SHA-256. Asset URLs
are built as `{base_url}/{path}`.

Public image URLs use stable `main/assets/...` paths without a game version.
The game version remains metadata, and `v<game-version>` tags are retained only
for rollback. Consumers detect rare same-path changes through the manifest hash.

## Updating

The scheduled workflow checks APKPure every six hours. It performs a complete
tracker extraction only when the game version changes, rebuilds the critical
set, commits it, and creates a rollback `v<game-version>` tag. When R2 Actions
variables and secrets are configured, it also incrementally publishes the
stable paths to Cloudflare R2. Unchanged SHA-256 objects are skipped and stale
objects are retained.
